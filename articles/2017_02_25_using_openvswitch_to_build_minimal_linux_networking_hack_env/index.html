Using OpenvSwitch to build minimal linux networking hack environment

2017/02/25

Categories: kernel Tags: linux kernel network openvswitch

Create Openvswitch bridge and two tap devices for vm

ip tuntap add name tap01 mode tap
ip tuntap add name tap02 mode tap

ip link set tap01 up
ip link set tap02 up

ip link set tap01 address 12:34:56:78:9a:01
ip link set tap01 address 12:34:56:78:9a:02

ovs-vsctl add-br br0
ovs-vsctl add-port br0 tap01
ovs-vsctl add-port br0 tap02

Start the two vms and config network

configure vm01

  • start vm01
qemu-system-x86_64 -m 2048M -enable-kvm \
-nographic \
-kernel ./bzImage -initrd ./initramfs \
-append  "console=ttyS0 quiet" \
-netdev tap,id=net1,ifname=tap01,script=no,downscript=no  \
-device e1000,netdev=net1,mac='12:34:56:78:9a:01'
  • configure vm01 network
ip link set eth0 up
ip addr add 192.168.0.1/24 dev eth0

configure vm02

  • start vm02
qemu-system-x86_64 -m 2048M -enable-kvm \
-nographic \
-kernel ./bzImage -initrd ./initramfs \
-append  "console=ttyS0 quiet" \
-netdev tap,id=net2,ifname=tap02,script=no,downscript=no  \
-device e1000,netdev=net2,mac='12:34:56:78:9a:02'
  • configure vm02 network
ip link set eth0 up
ip addr add 192.168.0.2/24 dev eth0

Check the vms network connection

vm01 ping vm02

ping 192.168.0.2
