Using OpenvSwitch to build minimal linux networking hack environment

Create Openvswitch bridge and two tap devices for vm

ip tuntap add name tap1 mode tap
ip tuntap add name tap2 mode tap

ip link set tap1 up
ip link set tap2 up

ovs-vsctl add-br br0

ovs-vsctl add-port br0 tap1

ovs-vsctl add-port br0 tap2

Start the two vms and config network

configure vm01

  • start vm01
qemu-system-x86_64  -nographic \
-kernel ./bzImage -initrd ./initramfs \
-append  "console=ttyS0 quiet" \
-netdev tap,id=net1,ifname=tap1,script=no,downscript=no  \
-device e1000,netdev=net1,mac='9e:11:24:5c:35:89'
  • configure vm01 network
ip link set eth0 up
ip addr add 192.168.0.1/24 dev eth0

configure vm02

  • start vm02
qemu-system-x86_64  -nographic \
-kernel ./bzImage -initrd ./initramfs \
-append  "console=ttyS0 quiet" \
-netdev tap,id=net2,ifname=tap2,script=no,downscript=no  \
-device e1000,netdev=net2,mac='66:5f:30:f4:24:be'
  • configure vm02 network
ip link set eth0 up
ip addr add 192.168.0.2/24 dev eth0

Test the vms network connection

vm01 ping vm02

ping 192.168.0.2
