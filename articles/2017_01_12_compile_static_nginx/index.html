Compile static nginx

2017/01/12

Categories: nginx Tags: nginx

Get nginx source code from officall website

wget http://nginx.org/download/nginx-1.10.2.tar.gz

Untar it to specific directory

tar zxf nginx-1.10.2.tar.gz -C /src

Go to the /src, and execute ./configure

cd /src/nginx-1.10.2

./configure --prefix=/src/nginx/ \
--with-cc-opt="-static -static-libgcc" \
--with-ld-opt="-static"

Build it with make -$(nproc)

After analyzing the output, we got common undefined reference error. But actually the system have the pthread library installed. The real cause is the link order.

Solution - change the objs/Makefile to adjust the link order

Get the following line in the file:

	-static -ldl -lpthread -lcrypt -lpcre -lcrypto -lcrypto -lz

and change it to:

	-static -ldl -lcrypt -lpcre -lcrypto -lcrypto -lz -lpthread

The reason is that function sljit_generate_code in pcre library need pthread library to resolve undfined symbol pthread_mutex_unlock.

Build again:

make -j$(nproc)

and the right result:

-static -ldl -lcrypt -lpcre -lcrypto -lcrypto -lz -lpthread
